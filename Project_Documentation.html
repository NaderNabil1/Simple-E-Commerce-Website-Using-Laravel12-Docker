Project Overview

This is a comprehensive Laravel-based E-Commerce Content Management System (CMS) designed for modern online businesses. Built with cutting-edge web technologies, the application provides a powerful admin dashboard for complete e-commerce management, including product catalog, order processing, user management, and inventory control.

Key Highlights: This platform features a robust API-first architecture, secure authentication with Laravel Sanctum, real-time order tracking, automated inventory management, and a responsive admin interface that works seamlessly across all devices.

Local Development Setup

1Clone the Repository

git clone <repository-url>
cd demo

2Install PHP Dependencies

composer install

3Install Node.js Dependencies

npm install

4Environment Configuration

Create a .env file in the project root:

APP_NAME="Laravel E-Commerce CMS"
APP_ENV=local
APP_KEY=
APP_DEBUG=true
APP_URL=http://localhost:8000

LOG_CHANNEL=stack
LOG_DEPRECATIONS_CHANNEL=null
LOG_LEVEL=debug

# Database Configuration (Choose one)
# For SQLite (Default)
DB_CONNECTION=sqlite
DB_DATABASE=/absolute/path/to/database/database.sqlite

# For MySQL
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=demo
DB_USERNAME=root
DB_PASSWORD=

# Mail Configuration
MAIL_MAILER=smtp
MAIL_HOST=mailpit
MAIL_PORT=1025
MAIL_USERNAME=null
MAIL_PASSWORD=null
MAIL_ENCRYPTION=null
MAIL_FROM_ADDRESS="hello@example.com"
MAIL_FROM_NAME="${APP_NAME}"

# Cache Configuration
CACHE_DRIVER=file
FILESYSTEM_DISK=local
QUEUE_CONNECTION=sync
SESSION_DRIVER=file
SESSION_LIFETIME=120

# Redis Configuration (Optional)
REDIS_HOST=127.0.0.1
REDIS_PASSWORD=null
REDIS_PORT=6379

5Generate Application Key

php artisan key:generate

6Database Setup

For SQLite (Recommended for Development)

# Create SQLite database file
touch database/database.sqlite

# Run migrations
php artisan migrate

# Seed the database (optional)
php artisan db:seed

For MySQL

# Create database
mysql -u root -p
CREATE DATABASE demo;

# Run migrations
php artisan migrate

# Seed the database (optional)
php artisan db:seed

7Create Storage Link

php artisan storage:link

8Build Frontend Assets

# For development
npm run dev

# For production
npm run build

9Start Development Server

# Start Laravel development server
php artisan serve

# Or use the development script (includes queue worker and logs)
composer run dev

Note: The application will be available at http://localhost:8000

Docker Setup

Prerequisites

  • Docker Desktop installed and running
  • Docker Compose v2.0+

1Environment Configuration for Docker

Create a .env file with Docker-specific settings:

APP_NAME="Laravel E-Commerce CMS"
APP_ENV=local
APP_KEY=
APP_DEBUG=true
APP_URL=http://localhost:6162

# Database Configuration for Docker
DB_CONNECTION=mysql
DB_HOST=db
DB_PORT=3306
DB_DATABASE=demo
DB_USERNAME=demo
DB_PASSWORD=secret

# Mail Configuration
MAIL_MAILER=smtp
MAIL_HOST=mailpit
MAIL_PORT=1025
MAIL_USERNAME=null
MAIL_PASSWORD=null
MAIL_ENCRYPTION=null
MAIL_FROM_ADDRESS="hello@example.com"
MAIL_FROM_NAME="${APP_NAME}"

# Cache Configuration
CACHE_DRIVER=file
FILESYSTEM_DISK=local
QUEUE_CONNECTION=sync
SESSION_DRIVER=file
SESSION_LIFETIME=120

2Build and Start Containers

# Build and start all services
docker-compose up -d --build

# View logs
docker-compose logs -f

3Install Dependencies Inside Container

# Install PHP dependencies
docker-compose exec app composer install

# Install Node.js dependencies
docker-compose exec node npm install

# Build frontend assets
docker-compose exec node npm run build

4Application Setup

# Generate application key
docker-compose exec app php artisan key:generate

# Run database migrations
docker-compose exec app php artisan migrate

# Seed database (optional)
docker-compose exec app php artisan db:seed

# Create storage link
docker-compose exec app php artisan storage:link

5Access the Application

  • Web Application: http://localhost:6162
  • Mailpit (Email Testing): http://localhost:8025
  • MySQL Database: localhost:3306

Docker Services

The Docker setup includes the following services:

  1. app: PHP-FPM container with Laravel application
  2. web: Nginx web server
  3. db: MySQL 8.0 database
  4. mailpit: Email testing tool
  5. node: Node.js container for frontend builds

Useful Docker Commands

# Stop all services
docker-compose down

# Stop and remove volumes
docker-compose down -v

# Rebuild specific service
docker-compose build app

# Execute commands in container
docker-compose exec app php artisan migrate
docker-compose exec node npm run dev

# View service logs
docker-compose logs app
docker-compose logs web
docker-compose logs db

  • View Admins: List all admin accounts
  • Add Admin: Create new admin accounts
  • Edit Admin: Modify admin information
  • Delete Admin: Remove admin accounts

  • View Categories: List all product categories
  • Add Category: Create new categories
  • Edit Category: Modify category information
  • Delete Category: Remove categories

5. Product Management (/admin/products)

  • View Products: List all products with filtering options
  • Add Product: Create new products with images and details
  • Edit Product: Modify product information
  • Delete Product: Remove products

6. Order Management (/admin/orders)

  • View Orders: List all orders with status filtering
  • Edit Order: Modify order details and status
  • Delete Order: Remove orders

7. Admin Management (/admin/employees)

  • View Employees: List all employee accounts
  • Add Employee: Create new employee accounts
  • Edit Employee: Modify employee information
  • Delete Employee: Remove employee accounts

API Documentation

API Base URL: http://localhost:8000/api (local) or http://localhost:6162/api (Docker)

Authentication: Bearer Token (Laravel Sanctum)

Content-Type: application/json

Authentication Endpoints

Register User

Endpoint: POST /api/register

Description: Register a new user account

Request Body:

{
    "name": "John Doe",
    "email": "john@example.com",
    "password": "password123",
    "password_confirmation": "password123"
}

Response (201):

{
    "user": {
        "id": 1,
        "name": "John Doe",
        "email": "john@example.com",
        "role": "user"
    },
    "token": "1|abc123..."
}

Login User

Endpoint: POST /api/login

Description: Authenticate user and get access token

Request Body:

{
    "email": "john@example.com",
    "password": "password123"
}

Response (200):

{
    "token": "1|abc123...",
    "user": {
        "id": 1,
        "name": "John Doe",
        "email": "john@example.com",
        "role": "user"
    }
}

Logout User

Endpoint: POST /api/logout

Description: Revoke current access token

Headers: Authorization: Bearer {token}

Response (200):

{
    "message": "Logged out"
}

Database Models & Schema

Model Relationships Overview

The application uses Laravel Eloquent ORM with well-defined relationships between models. Here's the complete database structure with all relationships and attributes.

User Model

Table: users

Relationships:

  • hasMany(Order::class) - User orders
  • hasMany(Order::class, 'assigned_to') - Assigned orders

Fillable: name, email, password, role

Special Features:

  • Auto-assigns 'admin' role to first user
  • isAdmin() method for role checking
  • Laravel Sanctum API tokens

Category Model

Table: categories

Relationships:

  • hasMany(Product::class) - Category products

Fillable: name, status

Features:

  • Product categorization
  • Status management (active/inactive)

Product Model

Table: products

Relationships:

  • belongsTo(Category::class) - Product category
  • hasMany(OrderItem::class) - Order items

Fillable: name, description, category_id, price, sale_price, sale_end_date, stock_quantity, status

Features:

  • Sale pricing with end dates
  • Stock quantity management
  • Product status control

Cart Model

Table: carts

Relationships:

  • belongsTo(User::class) - Cart owner
  • hasMany(CartItem::class) - Cart items

Fillable: user_id, status

Features:

  • User-specific carts
  • Status management (active/completed)

CartItem Model

Table: cart_items

Relationships:

  • belongsTo(Cart::class) - Parent cart
  • belongsTo(Product::class) - Product details

Fillable: cart_id, product_id, quantity

Features:

  • Quantity management
  • Product association

Order Model

Table: orders

Relationships:

  • belongsTo(User::class) - Order customer
  • belongsTo(User::class, 'assigned_to') - Assigned employee
  • hasMany(OrderItem::class) - Order items

Fillable: user_id, assigned_to, total_price, total_old_price, discount, quantity, status, order_code, name, email

Features:

  • Unique order codes
  • Price tracking (original vs sale)
  • Employee assignment
  • Status workflow management

OrderItem Model

Table: order_items

Relationships:

  • belongsTo(Order::class) - Parent order
  • belongsTo(Product::class) - Product details

Fillable: order_id, product_id, quantity, price, old_price, total_price, total_old_price, discount

Features:

  • Detailed price tracking
  • Discount calculations
  • Quantity management

Database Schema Details

Table Key Fields Relationships Purpose
users id, name, email, role, password Orders (1:many), Assigned Orders (1:many) User management and authentication
categories id, name, status Products (1:many) Product categorization
products id, name, price, sale_price, stock_quantity Category (many:1), OrderItems (1:many) Product catalog and inventory
carts id, user_id, status User (many:1), CartItems (1:many) Shopping cart management
cart_items id, cart_id, product_id, quantity Cart (many:1), Product (many:1) Cart item details
orders id, user_id, order_code, total_price, status User (many:1), OrderItems (1:many) Order management and tracking
order_items id, order_id, product_id, quantity, price Order (many:1), Product (many:1) Order item details and pricing
